# ZapretMac

> Обход блокировок YouTube на macOS — графическая оболочка над проверенным движком zapret.

---

## Зачем?

Я устал зависеть от чужих сервисов.

VPN, прокси, расширения в браузере, «волшебные» кнопки в Telegram — всё это работает через чей-то чужой сервер, чужие правила и чужое настроение. Сегодня открывается, завтра — нет. Сегодня бесплатно, завтра — подписка. А YouTube нужен просто посмотреть видео, без квеста «найди рабочий обход».

Долго искал нормальный **zapret для Mac**. На Windows и Linux его давно портировали, собирали, описывали. На macOS — тишина: либо полусырые скрипты, либо ничего. Готового, аккуратного, «поставил и забыл» — не нашлось.

Поэтому я решил **сделать это сам**. Взять проверенную идею zapret, адаптировать под macOS и завернуть в простое приложение, которое не стыдно показать на рабочем столе. Не ради хакерства — ради того, чтобы снова спокойно открыть YouTube у себя дома.

---

## Что это

**ZapretMac** — это не VPN и не прокси-сервис в облаке. Это локальная программа для macOS, которая:

- устанавливает и настраивает движок **zapret** (модуль **tpws**) на вашем Mac;
- перенаправляет нужный трафик через локальную обработку;
- подбирает рабочую **стратегию** обхода автоматически или вручную;
- показывает понятный статус и журнал — что происходит и почему.

Фокус сделан на **YouTube**. Остальной интернет по возможности не трогается.

---

## Как это работает (в общих чертах)

Программа состоит из двух слоёв: **красивого интерфейса** (Electron) и **системного движка** (zapret в `/opt/zapret`).

### 1. Установка и подготовка

При первом запуске ZapretMac (с правами администратора):

1. Копирует движок zapret в `/opt/zapret`.
2. Настраивает сетевой интерфейс (`IFACE_LAN`), список доменов YouTube и вспомогательные параметры.
3. Регистрирует системные службы для автозапуска и диагностики.
4. Подготавливает правила **PF** (встроенный фаервол macOS) для прозрачного перенаправления трафика.

После установки повторный ввод пароля нужен редко — основные операции выполняются через заранее настроенные права `sudo`.

### 2. Включение обхода

Когда вы нажимаете **«Включить»**:

1. **Подготовка сессии** — обновление скриптов, проверка PF, запуск `tpws`.
2. **Подбор стратегии** — если выбран режим «Авто», программа по очереди пробует пресеты (split, TLS, combo и др.) и проверяет доступность YouTube.
3. **Применение победителя** — рабочая стратегия сохраняется; при следующем запуске она проверяется первой.

Стратегии — это разные способы обработки сетевых соединений на уровне, который мешает провайдеру «узнавать» заблокированный трафик. Детали намеренно не раскрываются: чем меньше публичной кухни, тем дольше живёт обход.

### 3. Как трафик попадает в обход

ZapretMac использует **два режима** — в зависимости от того, что реально работает на вашей сети:


| Режим             | Когда               | Что делает                                                                                             |
| ----------------- | ------------------- | ------------------------------------------------------------------------------------------------------ |
| **Прозрачный PF** | PF-якоря загружены  | Браузер ходит как обычно; macOS перенаправляет трафик к нужным доменам через локальный `tpws`.         |
| **PAC + SOCKS**   | PF не проходит тест | Только домены YouTube/Google идут через локальный SOCKS `127.0.0.1:11080`; остальные сайты — напрямую. |


Второй режим появился не случайно: системный SOCKS на весь интернет ломает Firefox. PAC решает это — YouTube через обход, всё остальное без прокси.

### 4. Что происходит внутри tpws

`tpws` — локальный процесс на вашем Mac. Он:

- слушает перенаправленный трафик (прозрачный режим) и/или SOCKS-порт;
- применяет выбранную стратегию только к доменам из **списка** (YouTube, googlevideo, ytimg и т.д.);
- не отправляет ваш трафик на чужие серверы — всё остаётся на машине.

Дополнительно блокируется **QUIC** (UDP 443), чтобы браузер не обходил обработку через альтернативный протокол.

### 5. Выключение

**«Выключить»** останавливает `tpws`, снимает PF-правила, отключает PAC/SOCKS в настройках сети и возвращает систему в обычное состояние.

## Интерфейс

- **Кнопка питания** — включить / выключить обход.
- **Стратегия (ALT)** — авто-перебор или ручной выбор пресета.
- **Журнал** — что установилось, какая стратегия сработала, ошибки PF/VPN.
- **Автозапуск / автоподключение** — по желанию.

Иконка в меню-баре — фиолетовый кружок в палитре приложения (статус «работает / выключено»).

---

## Требования

- **macOS** (Apple Silicon или Intel)
- Права **администратора** (для PF, установки в `/opt/zapret`, `tpws`)
- Желательно **без активного VPN** при первой настройке PF (потом можно пробовать снова)

## Установка

### Из DMG (рекомендуется)

1. Откройте `ZapretMac-1.0.0-mac-arm64.dmg`.
2. Перетащите **ZapretMac** в папку «Программы».
3. При первом запуске: **ПКМ → Открыть → Открыть** (если macOS ругается на неподписанного разработчика).

### Подпись Apple

Без **Apple Developer ID** и **нотаризации** macOS может показывать предупреждение при установке — это нормально для локальной сборки.

---

## Использование

1. Запустите **ZapretMac**.
2. Нажмите **«Включить»**, введите пароль macOS **один раз**.
3. Дождитесь сообщения в журнале вроде `YouTube OK` или `PAC: YouTube → SOCKS`.
4. Откройте YouTube в браузере (лучше новая или приватная вкладка).
5. Для отключения — **«Выключить»**.

Если в журнале `VPN активен` — по возможности отключите VPN и перезапустите приложение.

---

## Что программа **не** делает

- Не меняет ваш IP и не прячет вас в другой стране (это не VPN).
- Не гарантирует обход всех сайтов — заточено под YouTube.
- Не обещает вечную работу: провайдеры меняют правила, стратегии приходится обновлять.
- Не отправляет трафик третьим лицам — всё локально.

## Структура проекта (для разработчиков)

```
ZapretMac/
├── src/main/          # Electron: логика, zapret-runtime, PAC, тесты
├── src/renderer/      # Интерфейс
├── resources/zapret-runtime/  # Движок zapret (tpws, PF, скрипты)
├── scripts/           # Сборка иконок, подпись
└── build/             # Entitlements для macOS
```

Сборка:

```bash
npm install
npm run build:dir    # .app в dist/
npm run build        # .app + .dmg + .zip
```

Подпись и нотаризация (при наличии Apple Developer):

```bash
./scripts/sign-and-notarize.sh
```
