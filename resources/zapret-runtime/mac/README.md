# zapret для macOS — короткая инструкция

Эта папка — обвязка вокруг оригинального `zapret` (бинарики и init-скрипты для мака
уже лежат рядом, в `binaries/mac64/` и `init.d/macos/`). Сам zapret поддерживает
macOS «из коробки», просто никто не написал понятного русского установщика для не‑айтишников. Эта папка решает именно это.

## Что это делает

На маке `zapret` работает через `tpws` — это маленький прозрачный прокси. Через
правила PF (системный фаервол macOS) трафик на 80/443 перенаправляется в `tpws`,
он «портит» начало TLS-рукопожатия так, чтобы провайдерский DPI не понял, что ты
открываешь YouTube, и пропустил соединение. Дальше всё работает нормально.

Применяется обход **только к доменам из списка**, остальной трафик идёт обычным
путём.

## Что нового (июнь 2026)

- **Тумблер без пароля.** Первый пункт меню — «⏻ Включить/Выключить обход»,
  и пароль он больше не спрашивает: `install.sh` кладёт правило в
  `/etc/sudoers.d/zapret`, разрешающее запускать только наши управляющие
  скрипты (они принадлежат root и недоступны на запись). Пароль остаётся
  только на установку и удаление.
- **Автопауза при VPN.** Демон `vpn-watch` следит за сетью: включился VPN —
  zapret сам встаёт на паузу (tpws гасится, PF-якоря снимаются), выключился —
  сам возобновляется. Ручные «Включить/Выключить» главнее автоматики.
  Отключить автопаузу: `sudo touch /opt/zapret/mac/vpn-watch.disabled`.
- **Окно при входе в систему.** Если zapret стартовал в фоне при загрузке,
  после логина автоматически открывается окно Zapret.app — видно, что обход
  включён, и его можно выключить одним кликом. Приложение теперь копируется
  в `/Applications` при установке.
- **Переключение стратегий из GUI.** Пункт «🧪 Стратегия обхода» — пять
  проверенных на macOS пресетов tpws (default / split-only / midsld / oob /
  hostcase); выбор записывается в конфиг и zapret перезапускается сам.

### Что было починено раньше

Если ты уже пользовался прошлой сборкой и ловил баги — вот что исправлено:

- **Зависало окно с неактивной кнопкой OK.** Причина — длинный текст в системном
  диалоге AppleScript (баг macOS). Теперь полный вывод пишется в лог-файл
  `/tmp/zapret-gui.log`, а в окне показывается только короткое резюме и кнопка
  **«Открыть лог»**. Подвиснуть больше нечему.
- **После «Остановить» статус оставался зелёным.** Новый `stop.sh` гарантированно
  выгружает демон, убивает все процессы `tpws` (по pid-файлам и по имени),
  ждёт их завершения и при необходимости добивает `SIGKILL`, а потом честно
  проверяет результат. Статус в окне теперь перепроверяется по факту.
- **Конфликт с VPN — zapret не запускался.** Из `start.sh` убран `set -e`
  (из-за него весь запуск падал, когда PF-команда спотыкалась о правила VPN).
  Теперь успех определяется по факту работы `tpws`, PF-патч умеет работать и с
  нестандартным `pf.conf` (переписанным VPN-клиентом), а сбой фаервола больше
  не валит запуск.

> ⚠️ **Важно:** правки GUI находятся в исходнике `ZapretControl.applescript`, а
> само `Zapret.app` — это уже скомпилированная версия. Чтобы исправления GUI
> заработали, приложение нужно **пересобрать** (`./build-app.sh`) и **переустановить**
> (кнопка «🔄 Установить / Переустановить» — она скопирует обновлённые скрипты в
> `/opt/zapret/mac/`). См. ниже.

### Применить исправления (по шагам)

Из папки `mac/` (там, куда распаковал/склонировал проект):

```bash
./build-app.sh
```

Затем открой `Zapret.app` → **🔄 Установить / Переустановить** (это разложит
обновлённые скрипты и PF-логику в `/opt/zapret`).

## Совместная работа с VPN (автопауза)

Раньше zapret и VPN конфликтовали из-за общих правил PF. Теперь это решается
автоматически: демон `vpn-watch` ловит включение VPN и **ставит zapret на
паузу**, а после выключения VPN — **возобновляет** его. Самому ничего нажимать
не надо. В статусе пауза видна как `⏸ на паузе — обнаружен VPN`.

`vpn-watch` распознаёт и системные VPN-профили (IKEv2/L2TP, через `scutil --nc`),
и приложения с туннелем поверх дефолтного маршрута (WireGuard, OpenVPN, Outline,
Amnezia, v2Ray-клиенты и т.п. — по интерфейсу `utun`/`ipsec`/`ppp`).

Нюансы:

- Ручные **«Включить» / «Выключить»** в GUI отменяют автопаузу — если ты сам
  включил обход при активном VPN, vpn-watch его уже не тронет до следующей
  смены состояния сети.
- Хочешь пользоваться обоими одновременно — включи обход вручную при активном
  VPN. Первый раз лучше прописать PF-якоря при выключенном VPN.
- Совсем отключить автопаузу: `sudo touch /opt/zapret/mac/vpn-watch.disabled`
  (вернуть — удалить этот файл). Лог автопаузы: `/tmp/zapret-vpnwatch.log`.

## Установка

Сначала нужно скачать проект. Либо архивом — зелёная кнопка **Code → Download
ZIP** на странице репозитория (или ZIP со страницы Releases), потом распаковать.
Либо командой `git clone` (см. ниже). Дальше — любой из трёх способов.

### Способ 1: двойным кликом (без терминала, проще всего)

1. Зайди в распакованную папку → в подпапку **`mac`**.
2. Дважды кликни по **`install.command`**.
   - Если macOS пишет «не удаётся открыть, неизвестный разработчик» —
     кликни правой кнопкой → **Открыть** → **Открыть**.
3. Откроется окно `Zapret.app` → нажми **🔄 Установить / Переустановить**,
   введи пароль администратора один раз.
4. Когда увидишь «готово» — открывай YouTube.

### Способ 2: одной командой в терминале

Скопируй целиком, вставь в **Terminal.app**, нажми Enter:

```bash
git clone https://github.com/glalker/Zapret-mac-m2-git.git && cd Zapret-mac-m2-git/mac && ./build-app.sh && open Zapret.app
```

Появится окно `Zapret.app` → **🔄 Установить / Переустановить** → пароль.

### Способ 3: чистый терминал, без GUI

Из папки `mac/`:

```bash
sudo ./install.sh
```

---

Дальше всё через окно приложения: **⏻ Включить / Выключить** (без пароля),
**🧪 Стратегия обхода**, **📊 Статус**, **📝 Список сайтов**, **🗑 Удалить**.
После установки zapret стартует автоматически при загрузке мака, а `Zapret.app`
копируется в `/Applications` — найдёт свои скрипты в `/opt/zapret/mac/` сам.

## Управление из терминала

Если предпочитаешь без GUI, все команды запускаются из папки `mac/` и требуют `sudo`:

| Команда                          | Что делает                              |
| -------------------------------- | --------------------------------------- |
| `sudo ./start.sh`                | Включить обход                          |
| `sudo ./stop.sh`                 | Выключить (трафик пойдёт напрямую)      |
| `sudo ./status.sh`               | Показать состояние, стратегию, VPN      |
| `sudo ./strategy.sh list`        | Список стратегий обхода                 |
| `sudo ./strategy.sh set <имя>`   | Сменить стратегию и перезапустить       |
| `sudo ./uninstall.sh`            | Снести zapret полностью                 |

## Список сайтов

Лежит в `/opt/zapret/ipset/zapret-hosts-user.txt`. Шаблон лежит здесь же — в
`zapret-hosts.txt`. По умолчанию там YouTube, Instagram, Facebook, Twitter/X,
LinkedIn, Discord, Reddit и ещё пара популярных сервисов.

Чтобы добавить свой сайт:

```bash
sudo nano /opt/zapret/ipset/zapret-hosts-user.txt
# дописал домен на новой строке, сохранил Ctrl+O, вышел Ctrl+X
sudo ./stop.sh && sudo ./start.sh
```

Поддомены подхватываются автоматически: добавил `example.com` — работает и
`www.example.com`, и `cdn.example.com`.

## Если не работает

1. Проверь статус:

   ```bash
   sudo ./status.sh
   ```

   Должны быть:
   - tpws запущен
   - PF включён
   - якоря zapret в `/etc/pf.conf`

2. Попробуй другую стратегию обхода — проще всего через GUI: пункт
   **«🧪 Стратегия обхода»**, выбери вариант, zapret перезапустится сам.
   Из терминала то же самое:

   ```bash
   sudo ./strategy.sh list           # посмотреть варианты
   sudo ./strategy.sh set oob         # например, попробовать oob
   ```

   Доступные пресеты (проверены на macOS, tpws v72.12):

   | Имя          | Что делает                                         |
   | ------------ | -------------------------------------------------- |
   | `default`    | сплит SNI `1,midsld` + `--disorder` (рекомендуется)|
   | `split-only` | только сплит, без disorder — если default рвёт связь|
   | `midsld`     | сплит по середине домена + disorder                |
   | `oob`        | сплит + out-of-band байт — против въедливых DPI     |
   | `hostcase`   | искажение Host (регистр+точка) + сплит — запасной  |

   Если хочется поэкспериментировать руками — открой `/opt/zapret/config`,
   найди блок `TPWS_OPT="..."` и впиши свои опции. Ещё пара вариантов под
   конкретных провайдеров:

   **Ростелеком / МТС:**

   ```
   TPWS_OPT="
   --filter-tcp=80 --methodeol --hostpad=1024 <HOSTLIST> --new
   --filter-tcp=443 --split-tls=sni --split-pos=1 --oob <HOSTLIST>
   "
   ```

   **Билайн / Мегафон:**

   ```
   TPWS_OPT="
   --filter-tcp=80 --methodeol --hostcase <HOSTLIST> --new
   --filter-tcp=443 --split-pos=1 --disorder --oob --oob-data=0xfe <HOSTLIST>
   "
   ```

   После правки конфига:

   ```bash
   sudo ./stop.sh && sudo ./start.sh
   ```

3. Подобрать стратегию автоматически: zapret умеет сам тестировать варианты.

   ```bash
   sudo /opt/zapret/blockcheck.sh
   ```

   Скрипт спросит домен (введи `www.youtube.com`), пройдётся по разным
   стратегиям и покажет, какие из них работают у твоего провайдера. Потом эту
   строку можно вписать в `TPWS_OPT`.

## Безопасность

- Скрипт правит системные файлы (`/etc/pf.conf`, `/Library/LaunchDaemons/`),
  поэтому требует `sudo`. Это нормально — иначе фаервол не подвинуть.
- **Тумблер без пароля сделан безопасно.** Правило в `/etc/sudoers.d/zapret`
  разрешает запускать без пароля **только** конкретные скрипты zapret из
  `/opt/zapret/mac/`. Эти скрипты и вся папка `/opt/zapret` принадлежат `root`
  и недоступны на запись обычному пользователю — то есть подменить их содержимое
  (чтобы протащить произвольную команду под root) нельзя. Это стандартный и
  безопасный паттерн «NOPASSWD на фиксированный список команд».
- `uninstall.sh` откатывает всё назад: чистит `/etc/pf.conf`, убирает якоря,
  удаляет launchd-юниты (zapret + vpn-watch), правило sudoers, агент
  автооткрытия, `/Applications/Zapret.app` и `/opt/zapret`.
- Никакой трафик никуда не отправляется, всё происходит локально на твоей
  машине. zapret не VPN — он не меняет твой IP, просто слегка ломает первый
  пакет TLS, чтобы DPI не смог распарсить SNI.
